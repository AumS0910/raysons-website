PERFORMANCE REQUIREMENTS
════════════════════════════════════════

Frame preloading strategy:
  Load first 20 frames immediately
  Load remaining frames in background chunks of 30
  Show progress indicator until first 20 loaded
  Never block scroll — show whatever frame is loaded

Memory management:
  Store frames as ImageBitmap not Image objects
  createImageBitmap() is faster to draw to canvas
  
Canvas optimization:
  Use requestAnimationFrame
  Only redraw when frameIndex changes
  Use will-change: transform on canvas element
  ctx.drawImage with correct cover calculation:
    const scale = Math.max(
      canvas.width / img.width,
      canvas.height / img.height  
    )
    const x = (canvas.width - img.width * scale) / 2
    const y = (canvas.height - img.height * scale) / 2
    ctx.drawImage(img, x, y, img.width*scale, img.height*scale)

GSAP specifics:
  Use GSAP ScrollTrigger with scrub: 0.5
  (not scrub: true — 0.5 gives slight lag 
   that feels more cinematic)
  
  ScrollTrigger.create({
    trigger: containerRef.current,
    start: 'top top',
    end: 'bottom bottom', 
    pin: true,
    scrub: 0.5,
    onUpdate: (self) => {
      const phase = self.progress < 0.5 ? 'assembly' : 'bore'
      const phaseProgress = self.progress < 0.5 
        ? self.progress * 2 
        : (self.progress - 0.5) * 2
      const frameIndex = Math.floor(phaseProgress * 239)
      setCurrentFrame(phase, frameIndex)
    }
  })

DEVICE DETECTION

const isMobile = () => {
  return window.innerWidth <= 1024 || 
    ('ontouchstart' in window) ||
    (navigator.maxTouchPoints > 0)
}

const isSlowConnection = () => {
  const conn = navigator.connection
  if(!conn) return false
  return ['slow-2g','2g','3g'].includes(conn.effectiveType)
}
